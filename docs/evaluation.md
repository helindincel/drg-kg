# Evaluation Framework

The DRG Evaluation Framework provides tools to measure the quality of Knowledge Graph extraction, graph construction, and reasoning.

## Core Concepts

- **Benchmark Dataset**: A collection of text and "gold standard" annotations (entities, relations, events, etc.).
- **Evaluation Report**: The output of running a benchmark, containing metrics like Precision, Recall, and F1.
- **Regression Comparison**: A tool to compare two reports and identify quality changes between versions or models.

## Metrics Supported

### Extraction Quality
- **Entity P/R/F1**: Measures accuracy of entity detection and optionally typing.
- **Relationship P/R/F1**: Measures accuracy of triple extraction.
- **Event P/R/F1**: Measures accuracy of complex event frames.

### Graph Structural Quality
- **Graph Coverage**: Percentage of gold entities/relations found in the predicted graph.
- **Graph Density**: Ratio of actual edges to possible edges.
- **Orphan Node Rate**: Percentage of nodes with no connected edges.

### Reasoning Quality
- **Inference Accuracy**: P/R/F1 for edges generated by the reasoning layer.

### Entity Resolution
- **Pairwise P/R/F1**: Accuracy of clustering entities into canonical IDs.
- **Community Quality**: Accuracy of graph community detection.

## CLI Usage

### Running a Benchmark

```bash
drg eval run examples/benchmarks/corporate_acquisition_benchmark.json -o reports/run_001.json
```

### Comparing Reports (Regression Testing)

```bash
drg eval compare reports/baseline.json reports/candidate.json -o reports/regression_report.md
```

## Creating Benchmark Datasets

Benchmark datasets are stored as JSON files. See `examples/benchmarks/corporate_acquisition_benchmark.json` for a complete example.

```json
{
  "name": "my_benchmark",
  "text": "...",
  "gold_entities": [...],
  "gold_relations": [...]
}
```

## Automated Testing

The evaluation framework is used in internal tests to ensure pipeline changes do not reduce quality. Run the tests with:

```bash
pytest tests/test_evaluation.py
```
